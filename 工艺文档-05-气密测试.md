# 🔧 工艺 5：气密测试 (Leakage/Air-tightness Test)

---

## 📄 5.1 URS — 气密测试设备技术规格书

### 1. 项目概况与总体要求

| 项目 | 说明 |
|------|------|
| **设备名称** | 自动化气密性检测设备 |
| **设备节拍 (C/T)** | 单件 ≤ 60 秒（含装夹、充气、保压、检测、判定） |
| **制程能力要求** | 泄漏率检测 **GR&R ≤ 10%** |

### 2. 核心工艺技术参数 (硬件要求)

| 参数 | 规格要求 |
|------|----------|
| **测试压力** | 10-500 kPa 可调 |
| **保压时间** | 1-300 秒可调 |
| **允许泄漏率** | ≤ 0.1 ml/min（精密级）/ ≤ 1 ml/min（标准级） |
| **测试介质** | 压缩空气（ISO 8573-1 Class 1）或氦气 |
| **压力传感器** | 高精度数字压力传感器（推荐：Setra / WIKA），精度 ≤ ±0.1% FS |
| **流量传感器** | 质量流量传感器（推荐：Brooks / Alicat），精度 ≤ ±0.5% |
| **密封工装** | 自动密封，密封力可控，密封圈材质：FKM/EPDM |

### 3. 控制策略与防错逻辑 (软件要求)

1.  **压力衰减法:** 充气→稳压→保压→检测压降，判定泄漏率。
2.  **流量法:** 直接测量泄漏流量，适用于高精度检测。
3.  **氦检漏法（可选）:** 使用氦气作为示踪气体，质谱仪检测，灵敏度 ≤ 10⁻⁶ mbar·L/s。
4.  **多腔体测试:** 支持同时测试多个腔体，独立判定。
5.  **温度补偿:** 自动温度补偿，消除环境温度影响。

### 4. 工业控制、MES与数据追溯

| 项目 | 规格要求 |
|------|----------|
| **PLC 控制系统** | 西门子 S7-1200 或 Beckhoff |
| **数据采集** | 高精度 AD 采集，采样率 ≥ 1kHz |
| **数据上传** | OPC UA |
| **数据内容** | 产品ID、泄漏率、压降值、测试压力、OK/NOK |
| **数据存储** | ≥100万条记录，支持趋势分析和 SPC |

### 5. EHS (环境、健康与安全)

| 项目 | 要求 |
|------|------|
| **压力安全** | 安全阀、压力释放装置，符合 GB/T 150 压力容器标准 |
| **防护罩** | 透明防护罩，联锁装置，测试过程中禁止开门 |
| **急停按钮** | 设备四角各一个 |
| **噪音等级** | ≤ 70 dB |

### 6. 设备验收标准 (FAT / SAT)

| 验收阶段 | 方法 | 合格标准 |
|----------|------|----------|
| **预验收 (FAT)** | 供应商场地使用标准漏孔验证 | 泄漏率测量误差 ≤ ±5% |
| **终验收 (SAT)** | 买方产线，使用标准漏孔 + GR&R 测试 | GR&R ≤ 10%，连续运行 24 小时稳定 |

## 📄 5.2 SOP — 气密测试标准作业指导书

### 1. 准备工作与 5S 要求

| 项目 | 要求 |
|------|------|
| **PPE** | 安全眼镜、劳保鞋 |
| **5S** | 工位整洁，工具定置定位 |
| **开班点检** | 使用标准漏孔验证设备精度，检查密封圈状态 |
| **首件检验** | 使用 OK 标准件 + NOK 标准件（带标准漏孔）验证设备判定 |

### 2. 标准操作步骤 (Step-by-Step)

| 步骤 | 动作描述 | 关键质量要点 (CTQ) / 防呆 |
|------|----------|--------------------------|
| **1. 扫码** | 扫描产品条码，系统自动调用测试程序 | 条码错误，程序锁定 |
| **2. 装夹** | 将产品放入测试工装，自动密封 | 密封面无异物，密封力到位 |
| **3. 启动** | 关闭防护门，按下启动按钮 | 防护门未关闭，设备无法启动 |
| **4. 测试** | 系统自动充气、稳压、保压、检测 | 观察压力曲线，无异常波动 |
| **5. 判定** | 系统自动判定 OK/NOK | OK 件取出；NOK 件隔离 |
| **6. 记录** | 系统自动生成测试报告 | 数据自动上传 MES |

### 3. 异常处理与升级流程 (Escalation)

| 级别 | 触发条件 | 处理措施 |
|------|----------|----------|
| **轻微报警** | 单次 NOK | 复位重测，确认产品密封性 |
| **严重报警** | 连续 3次 NOK，或设备压力异常 | **立即停止**，按急停，拉 Andon，通知设备工程师 + 工艺工程师 |

### 4. 首件检验规范

| 项目 | 要求 |
|------|------|
| **检验频次** | 每班开班前、换型后、设备重启后 |
| **检验工具** | OK 标准件、NOK 标准件（带已知泄漏率的标准漏孔） |
| **检验方法** | 使用标准件测试，确认设备判定结果正确 |
| **记录** | 填写首件检验记录表，班长签字确认 |

### 5. 条码追溯操作

| 项目 | 要求 |
|------|------|
| **扫码时机** | 每件产品测试前必须扫码 |
| **防呆措施** | 无条码、重码、条码模糊均无法启动设备 |
| **数据关联** | 系统自动将条码与测试数据关联上传 MES |

## 📄 5.3 PFMEA — 气密测试过程失效模式与影响分析

> 基于 AIAG-VDA 统一标准格式编制

| 过程步骤 | 潜在失效模式 (FM) | 潜在失效影响 (Effect) | 严重度 (S) | 潜在失效原因 (Cause) | 频度 (O) | 现行预防控制 (PC) | 现行探测控制 (DC) | 探测度 (D) | RPN | 建议优化措施 (Action) |
|----------|-------------------|-----------------------|------------|-----------------------|----------|-------------------|-------------------|------------|-----|----------------------|
| **10. 产品装夹** | 密封不良 | 产品泄漏，测试不准 | 7 | 密封圈老化，密封面异物 | 3 | 定期更换密封圈，装夹前清洁 | 密封压力监控，压力不稳定报警 | 3 | **63** | 引入密封圈寿命计数器 |
| **20. 充气保压** | 接口泄漏 | 测试不准，假阳性 | 5 | 接头未插到位，管路松动 | 3 | 接头扭矩标准，管路定期检查 | 压力稳定性监控，泄漏率超限报警 | 3 | **45** | 引入管路泄漏自检功能 |
| **30. 检测判定** | 测试腔体泄漏 | 设备故障，效率降低 | 6 | 腔体密封面磨损，螺栓松动 | 2 | 定期更换密封件，螺栓扭矩校验 | 腔体压力监控，泄漏率超限报警 | 3 | **36** | 增加腔体泄漏自检程序 |
