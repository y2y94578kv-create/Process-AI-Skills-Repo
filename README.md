## 📋 项目简介

本仓库提供一套 **LangGPT 结构化 System Prompt 模板** + **9 份制造工艺深度文档**，覆盖设备招标、车间操作、质量分析全场景。每份文档包含可直接使用的 **URS（用户需求规格书）**、**SOP（标准作业指导书）** 和 **PFMEA（过程失效模式与影响分析）**，颗粒度达到工业量产标准。

## 🚀 V3.0 终极落地版 升级亮点

### URS（设备技术规范 — 招标级）
每份 URS 直接可用于向设备供应商招标，包含：

| 高阶要素 | 内容 |
|----------|------|
| **控制与电气品牌清单** | PLC（西门子 S7-1511F / 三菱 FX5U-32MT）、气动元件（SMC / Festo）、传感器（Keyence / Cognex）、安全光栅（SICK deTec4 / Keyence GL-R）、安全继电器（Pilz PNOZ / SICK RB12）、三色报警灯（Patlite / Murrelektronik）、低压电气（施耐德 / 魏德米勒） |
| **MES 信息化握手协议** | OPC UA 强制，12 项上传字段（SN/时间/判定/工艺值/曲线路径/设备ID/操作工ID/计数等），失败重试 3 次 + SQLite 本地缓存 ≥10000 条 |
| **EHS 与人体工程学** | 安全光栅 ≤10ms、双手启动间距 ≥260mm（硬线回路）、急停 5 点位（EN ISO 13850）、三色灯 + 蜂鸣器 ≥85dB、上料高度 800±50mm、操作力 ≤30N |
| **FAT/SAT 验收量化指标** | **GR&R %SV ≤ 10%** + **Cpk ≥ 1.67** + **OEE ≥ 95%** + **MTBF ≥ 5000 h** + **MTTR ≤ 30 min**（含验收公式与方法） |

### SOP（标准作业指导书 — 车间受控级）
每份 SOP 直接下发车间工位，操作工按步骤执行即可：

| 高阶要素 | 内容 |
|----------|------|
| **开班防呆验证** | 高限件（×1.2）3 次 100% 报警 + 低限件（×0.8）3 次 100% 通过 + 安全光栅/急停每日测试 |
| **动作拆解表** | 5-7 步 × 3 列：**动作拆解 → CTQ 质量关注点 → 异常防呆**，关键步骤含 `[插入图片]` 占位符 |
| **首件检验** | OK/NOK 标准件 × 3 次全项验证，不合格 → **立即锁定设备** |
| **Escalation 升级矩阵（4 级）** | L1 操作工（单件 NOK→2min）→ L2 班长（**连续 3 件 NOK→强制停机**→5min）→ L3 PE/QE（连续 5 件→30min）→ L4 工程经理（批量≥20 件→停线+CAPA） |

### PFMEA（过程失效模式分析 — AIAG-VDA 标准）

| 高阶要素 | 内容 |
|----------|------|
| **PC vs DC 严格分列** | 预防控制(PC) = 防呆设计/参数硬约束；探测控制(DC) = 传感器/视觉/设备检测 — 二者绝不混淆 |
| **4M1E 根因分析** | 根因深入物理端，标注 **人(Man)-机(Machine)-料(Material)-法(Method)-环(Environment)** |
| **行动优先级 AP** | H(高) / M(中) / L(低) 三级 + 责任方 + 处理时限要求 |

## 🚀 快速开始

### 1. 使用 LangGPT 模板

复制 `langppt-process-engineer.md` 中的 System Prompt 到你的 AI 助手（如 ChatGPT、Claude 等）中，即可开始生成工艺文档。

### 2. 交互流程

```
用户 → "请帮我生成一份拧紧工艺的 URS"
  ↓
AI → 确认文档类型：URS ✅
  ↓
AI → 确认工艺类型：拧紧工艺 ✅
  ↓
AI → 根据知识库生成完整 URS 文档
```

### 3. 直接使用工艺文档

每份 V3.0 `工艺文档-XX-工艺名称.md` 已包含完整的 URS + SOP + PFMEA，可直接用于：
- 📤 **URS** → 发给设备供应商作为招标技术附件
- 🏭 **SOP** → 打印塑封后悬挂于车间工位
- 📊 **PFMEA** → 提交客户 PPAP 审核包 / 内部质量体系审核
