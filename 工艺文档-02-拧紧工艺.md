# 🔧 工艺 2：伺服拧紧工艺 (Tightening)

---

## 📄 2.1 URS — 拧紧设备技术规格书

### 1. 项目概况与总体要求

| 项目 | 说明 |
|------|------|
| **设备名称** | 多轴伺服拧紧专机 |
| **设备节拍 (C/T)** | 单件完成时间 ≤ 10 秒（含上下料、扫码、拧紧、判定） |
| **制程能力要求** | 关键特性（最终扭矩、转角）**Cpk ≥ 1.67**，测量系统分析 **GR&R ≤ 10%** |

### 2. 核心工艺技术参数 (硬件要求)

| 参数 | 规格要求 |
|------|----------|
| **拧紧枪类型** | 伺服电动拧紧枪（推荐：Atlas Copco / Ingersoll Rand / Bosch Rexroth） |
| **扭矩量程** | 0.5-100 Nm（根据螺栓规格选择） |
| **扭矩精度** | ≤ ±2% |
| **角度范围** | 0-1080° |
| **角度分辨率** | 0.1° |
| **多轴同步** | 支持 2-8 轴同步拧紧，各轴扭矩偏差 ≤ ±3% |
| **套筒接口** | 快换式套筒，支持 M4-M16 螺栓 |

### 3. 控制策略与防错逻辑 (软件要求)

设备软件必须具备"扭矩-角度曲线 (Torque-Angle Curve)"实时监控能力，并支持以下判定逻辑（Poka-Yoke）：

1.  **贴合点检测 (Thread Engage):** 扭矩从 0 突然上升，判定为贴合点。若贴合扭矩 > 设定阈值，判定为"螺纹有杂质"或"螺栓型号错误"。
2.  **包络线监控 (Envelope/Window):** 支持设定不少于 5 个评估窗口。扭矩-角度曲线必须 100% 穿过设定的通道，任何一点触碰边界，立即判定为 NOK。
3.  **最终扭矩与最终角度 (End Torque & Angle):** 到达目标扭矩时，旋转角度必须在工艺公差范围内（如 90° ± 10°）。
4.  **屈服点控制 (Yield Point Control):** 监控扭矩上升斜率，当斜率发生明显变化时判定为屈服点，停止拧紧。

### 4. 工业控制、MES与数据追溯

| 项目 | 规格要求 |
|------|----------|
| **PLC 控制系统** | 西门子 S7-1200/1500 或三菱 Q/iQ 系列 |
| **拧紧枪通讯** | Ethernet/IP 或 PROFINET |
| **扫码追溯** | Keyence/Cognex 固定式扫码枪，无条码/重码防呆 |
| **数据上传** | OPC UA 或 TCP/IP |
| **数据内容** | 产品条码、最终扭矩、最终转角、OK/NOK 结果、CSV 格式曲线数据 |
| **数据存储** | ≥100万条记录 |

### 5. EHS (环境、健康与安全)

| 项目 | 要求 |
|------|------|
| **安全光栅** | Sick / Omron，响应时间 ≤ 20ms |
| **双手启动** | 时间差 ≤ 0.5 秒 |
| **噪音等级** | ≤ 75 dB |
| **PPE** | 安全眼镜、防割手套、劳保鞋 |

### 6. 设备验收标准 (FAT / SAT)

| 验收阶段 | 方法 | 合格标准 |
|----------|------|----------|
| **预验收 (FAT)** | 供应商场地连续拧紧 300 件 | 零故障，良率 ≥ 99.8% |
| **终验收 (SAT)** | 买方产线，提供扭矩/角度标定证书 | 完成 GR&R 测试，连续运行 24 小时 Cpk ≥ 1.67 |

## 📄 2.2 SOP — 拧紧标准作业指导书

### 1. 准备工作与 5S 要求

| 项目 | 要求 |
|------|------|
| **PPE** | 安全眼镜（防飞溅）、防割手套、劳保鞋 |
| **5S** | 工位整洁，工具定置定位 |
| **开班点检** | 检查拧紧枪套筒是否磨损、线缆是否破损、校准状态 |
| **首件检验** | 使用"点检标准件（Master Part）"：标准螺栓（OK）+ NOK 螺栓，验证设备判定功能 |

### 2. 标准操作步骤 (Step-by-Step)

| 步骤 | 动作描述 | 关键质量要点 (CTQ) / 防呆 |
|------|----------|--------------------------|
| **1. 扫码** | 拿起待装配的零件，在扫码枪下扫描条码 | 听见"滴"声，屏幕显示条码变绿。若条码错误，设备锁定 |
| **2. 上料** | 将零件放入定位工装中，手动放入螺栓 | **注意防反：** 确认零件的定位特征朝向正确方向。工装有防反销 |
| **3. 启动** | 双手离开光栅区域，同时按下左、右两侧的绿色启动按钮 | 按下直到拧紧枪开始旋转后方可松手。若中途手伸入光栅，设备会报警急停 |
| **4. 判定** | 观察设备屏幕显示的扭矩-角度曲线和判定结果 | 屏幕显示绿色 **OK**，拧紧枪自动停止；显示红色 **NOK**，警报灯亮起 |
| **5. 下料** | 取出拧紧好的组件。OK 件放入右侧蓝筐；NOK 件放入左侧红筐 | **严禁将 NOK 件混入良品筐** |

### 3. 异常处理与升级流程 (Escalation)

| 级别 | 触发条件 | 处理措施 |
|------|----------|----------|
| **轻微报警** | 连续 ≤ 2次 NOK | 操作工自行复位，将不良品隔离登记，重新拿取新零件进行拧紧 |
| **严重报警** | 连续 3次 NOK 或异常异响 | **立即停止操作**，按下急停按钮，拍下屏幕报警界面，拉响车间 Andon（暗灯）系统，或对讲机呼叫**带班班长**及**工艺工程师（PE）** |
| **禁止事项** | — | **禁止操作工擅自打开电气柜或修改拧紧参数** |

### 4. 首件检验规范

| 项目 | 要求 |
|------|------|
| **检验频次** | 每班开班前、换型后、设备重启后 |
| **检验工具** | OK 标准螺栓、NOK 标准螺栓 |
| **检验方法** | 使用标准件拧紧，确认设备判定结果正确 |
| **记录** | 填写首件检验记录表，班长签字确认 |

### 5. 条码追溯操作

| 项目 | 要求 |
|------|------|
| **扫码时机** | 每件产品拧紧前必须扫码 |
| **防呆措施** | 无条码、重码、条码模糊均无法启动设备 |
| **数据关联** | 系统自动将条码与拧紧数据关联上传 MES |
